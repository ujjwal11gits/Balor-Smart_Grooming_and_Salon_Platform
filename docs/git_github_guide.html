Git & GitHub Guide

A practical, beginner-friendly manual using the Balor Salon project as reference.

📂 1. The Core Concept (Game Save System Analogy)

  • Working Directory: Your folder on your laptop (e.g. D:\Balor-Smart_Grooming_and_Salon_Platform).
  • Staging Area: Picking which files to include in your next save point (git add).
  • Commit: Creating a permanent save point in history with a label (git commit).
  • Push: Uploading your local save files to the cloud on GitHub (git push).
  • Pull: Downloading someone else's edits from GitHub to your laptop (git pull).

🌿 2. Working with Branches (Parallel Workspaces)

Why we use branches:

Branches let you code new features (like a payment module) safely in a parallel space without breaking the working code on your main branch.

# Check current active branch
git branch

# Create and switch to a new branch 'add-payments'
git checkout -b add-payments

# Switch back to 'main'
git checkout main

📤 3. Daily Command Loop

Run these commands on your computer to save work and sync it to GitHub:

# Step A: Check changed files
git status

# Step B: Stage changes
git add .

# Step C: Commit save state
git commit -m "feat: added barber shop details page"

# Step D: Sync online
git push origin main

🤝 4. Merging Branches

When your feature on add-payments works perfectly and you want to combine it with main:

git checkout main
git merge add-payments
git push origin main

💥 5. Merge Conflicts (How to Fix)

What is a Merge Conflict?

A conflict happens when the exact same line of code in the same file is changed differently in two places, and Git doesn't know which version to keep.

Balor Project Example:

Inside backend/utils/mailer.js, you changed line 10 to Gmail on main, but someone changed line 10 to Brevo on the feature branch. During merge, Git halts and inserts conflict indicators:

<<<<<<< HEAD
const host = 'smtp.gmail.com'; // Your version on main
=======
const host = 'smtp.brevo.com';  // Version from the add-emails branch
>>>>>>> add-emails

How to Resolve:

  1. Open the file in **VS Code**.
  2. Use the GUI buttons to select *"Accept Current Change"* (Gmail), *"Accept Incoming"* (Brevo), or edit it manually to choose one and delete the indicator lines (<<<<<<<, =======, >>>>>>>).
  3. Run the commands to save: 
    git add backend/utils/mailer.js
git commit -m "fix: resolved mailer host merge conflict"
git push origin main

🛡️ 6. Undoing and Stashing

  • Undo local unsaved edits in mailer.js: 
    git checkout -- backend/utils/mailer.js
  • Temporarily shelve your current edits to fix a bug on another branch: 
    git stash       # saves edits and cleans workspace
# ... do other work ...
git stash pop   # brings back your shelved edits
