Vercel & Render Deployment Guide

Step-by-step documentation for deploying frontend on Vercel and backend API on Render.

🛠️ Prerequisites

  • Accounts on **Vercel** and **Render** connected to your GitHub profile.
  • Your codebase uploaded to a GitHub repository.
  • A hosted database connection string (e.g. MongoDB Atlas).

🎨 Phase 1: Deploying Frontend to Vercel

Step 1: Configure Client-Side Routing

Why we do this:

React uses client-side routing. We write a vercel.json file to tell Vercel to route all server URL requests back to index.html so that page reloads on paths like /dashboard do not throw 404 errors.

Create a file named vercel.json inside your frontend/ root folder:

{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}

Step 2: Connect and Configure Build on Vercel

  • Go to the Vercel Dashboard, click Add New → Project.
  • Import your Git repository.
  • Under **Project Configuration**:
    • Framework Preset: Vite
    • Root Directory: frontend
    • Keep Build/Install commands as default.

Step 3: Define Environment Variables on Vercel

Why we do this:

We tell our React files where our Render API backend lies.

  • Add the key-value pair under Environment Variables:
    • Key: VITE_API_URL
    • Value: https://barber-app-backend.onrender.com (your Render Web Service URL)
  • Click Deploy.

⚙️ Phase 2: Deploying Backend to Render

Step 4: Create Render Web Service

  • On Render dashboard, click New + → Web Service.
  • Connect your GitHub repository.

Step 5: Configure Build Settings

Why we do this:

Instructs Render on how to navigate to the Node folder, install packages, and boot the server.

  • Root Directory: backend
  • Runtime: Node
  • Instance Type: Free
  • Build Command: npm install
  • Start Command: npm start (triggers node server.js)

Step 6: Add Backend Env Variables

Why we do this:

Connects your MongoDB Atlas database and sets up your sessions, tokens, and email keys securely.

  • Add the following keys under Environment Variables:
    • PORT → 5000
    • MONGO_URI → mongodb+srv://competeujjwal_db_user:...@cluster0...
    • JWT_SECRET → MY_JWT_SECRET
    • JWT_REFRESH_SECRET → your_refresh_secret
    • FRONTEND_URL → https://barber-app-frontend.vercel.app (your Vercel App URL)
    • SMTP_USER / SMTP_PASS (for Gmail credentials)
  • Click Create Web Service.

🔄 CI/CD Update Process

Why it's convenient:

Whenever you push local changes to GitHub, Vercel and Render automatically detect the changes, pull the updates, rebuild, and redeploy your live site automatically.

# Make changes on your laptop, then run:
git add .
git commit -m "update project details"
git push origin main
